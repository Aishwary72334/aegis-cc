# AEGIS Secure OS - Monorepo

AEGIS is a secure productivity workspace that allows individuals, managers, and assistants to collaborate safely. It includes integrated scheduling, task management, goals tracking, notes, analytics, and robust role-based access delegation.

## Features

- **Personal & Shared Workspaces**: Toggle seamlessly between your personal workspace and workspaces delegated to you.
- **Granular Delegation Control**: Invite delegates with specific roles:
  - **Viewer**: Read-only access to tasks, events, goals, notes, and projects.
  - **Assistant/Manager**: Collaborative write access. (Only workspace owners can access profile credentials or revoke/grant access).
- **Consolidated Production Build**: Backend Express API server compiles and serves static frontend assets (`/dist`) dynamically under production mode, ensuring single-host compatibility.

---

## Folder Structure

- `/src`: Frontend React + Vite SPA (using Lucide icons and Recharts analytics).
- `/server`: Node/Express Backend API server and security middlewares.
- `/dist`: Production-compiled frontend assets (generated by `npm run build`).

## Local Development Setup

### 1. Prerequisites
- Node.js (v18+)
- A Supabase Project instance

### 2. Configure Environment Variables
Create a `.env` file in the `server` directory using the structure detailed in `server/.env.example`.
Create a `.env` file in the root directory using the structure detailed in `.env.example`.

### 3. Database Migration
Paste and run the SQL instructions located in:
- `supabase_schema.sql` (base schema)
- `migration_v2.sql` (workspace sharing & delegation upgrades)

### 4. Running the App
Install root dependencies and start client + server concurrently:
```bash
npm install
npm run dev
```

## Production Deployment

This project is configured to run on a single host (such as **Render** or **Railway**).

1. **Build Command**: `npm run build` (builds client assets and installs server modules).
2. **Start Command**: `npm start` (launches production Express server).
3. **Environment variables needed**:
   - `NODE_ENV=production`
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (kept private, used for delegation lookup)
